Introduction

The fact you're reading this introduction means you've hopefully managed to download and install the TrapKAT editor successfully.

This help page doesn't go into all the details of what all the settings on your TrapKAT mean - you still need the manual for that.

If this page doesn't answer a question about the editor, please feel free to post a bug! Use the Sourceforge Project Tracker, once it's available.

The editor is just that. It can load and edit "data dumps" produced by your TrapKAT. It cannot receive dumps from or transmit dumps to it, though. For that, I recommend MIDI-OX. There's a brief summary of how to use it below.

The software includes an auto-update system. This is mainly useful because of the immature nature of the program. I'm expecting to be releasing bugfixes and updates on a very regular basis. More on this below, too.

But first, the important bit..!

Loading files, editing files

When you first start the editor, you're presented with a set of values which may well end up looking like the factory defaults but are currently fairly random. You'll want to load something more useful.

The File menu contains the usual options:

  • New to get back to this initial state
  • Open to pick a data dump to edit
  • Save type to save an edited data dump
  • Save As type to save a data dump under a new name
  • Close or Exit to quit the program

The type represents one of

  • All memory
  • Global
  • Kit

reflecting the different data dumps that the TrapKAT can send. Open understands the different types and remembers the filename, so if you just do Ctrl-S you will save the appropriate kind of dump.

You can load a Kit dump into any kit location. Saving a kit dump will give it the kit number of the location it is saved from. When you send the dump to the TrapKAT, it will load into that kit number.

To get you going (and in case of emergency...), I include TKReInit.syx in the program folder. This is the state of my TrapKAT after doing a Reinitialise. Currently, it's a better starting point than you get with File->New, so load that up and have a look around the editing screens.

There are two main screens:

  1. Kits & Pads
  2. Global

Kits & Pads

Obviously, this splits into two parts:

  1. Kit Editor
  2. Pad Editor

Kit Editor

Hopefully all this is pretty much self-explanatory. The Select Kit: drop down lets you select which of the 24 user kits you want to edit. The kit names are listed as a quick reminder, if you've been naming them.

The Various (and Var.) check boxes around the form indicate whether or not individual pad settings apply. You have to check the box to enable the equivalent Pad Editor input.

Gate times can be entered freeform but will round down to the nearest available setting. The drop down list can be used to pick Latch mode, Infinite or Roll Mode.

Pad Editor

The Select Pad: drop down lets you select which of the 24 main pads you want to edit. Pad 25 is the bass/kick pedal. Pad 26 is the "chick" sound. I'm not sure what 27 and 28 are. Maybe Hat Note Mode stuff??

Notes 1 to 6 are used with the advanced Curve settings. Note 2 is the Open sound on a Hihat pad (a pad with Flag 7 checked). You can enter notes as MIDI note numbers (0 to 127) or musical notes like C#4. C4 is MIDI note 60. (I might make a setting to allow C3 to be MIDI note 60 at some point..!) The drop down allows the non-note value settings to be chosen.

As mentioned, Flag 7 indicates a pad is a Hihat pad. It can be fiddled with independently from the Hihat Pads: for the kit... so take care! Flag 6 is documented as Motif - maybe other uses for those six notes..?

Global

The Global editor is currently a little rough'n'ready. I'll be making it prettier shortly - i.e. putting meaningful dropdown boxes in appropriate locations. For the time being, just use it carefully...

Auto-updates

The editor has a built-in update facility. When you first run it (and after each update - might take that out...), you'll be asked whether you want the program to check for new versions automatically. It will check at most once per day. It can't check if you don't run it, either.

If you say No, you can check manually using Help->Check for update... or switch to automatic checking with Help->Check automatically. That menu option will be checked to indicate it's selected.

Help->About gives you details of your current version and what your update settings are. You'll notice the Update URL: field - that's where the software goes to find out if there is a new version.

Getting data in and out of your TrapKAT

As mentioned in the Introduction, the editor provides no means for receiving data dumps from or sending data dumps to the TrapKAT. I recommend MIDI-OX.

It can be a painless process. Receiving data is generally not a problem at all. However, sending data can take some gentle tuning to get it to work. The reason is that modern PCs send data much, much faster than some hardware was expecting to receive it at (even within the MIDI spec). With a MIDI data dump, the receiver is expected to handle 21,364 bytes arriving near enough at the same time. No problem for a PC, as I say. But it helps to slow things down for the TrapKAT.

At the bottom of the editor window I had put a line of text as a reminder of the settings in MIDI-OX that work for me.

Right, I better explain MIDI-OX, then!

Basic set up

Get wired up simply. MIDI Out from the TrapKAT to MIDI In on your PC. MIDI Out from your PC to MIDI In on the TrapKAT. (You might want to have other devices on the way in either direction but it's safest to start with these out of the way. Then, when you reconnect them, you can tell if they cause a problem.)

Run MIDI OX and make sure you have it receiving normal data from your TrapKAT. In MIDI-OX, go to Options->MIDI Devices.... Make sure you have your MIDI In port selected under MIDI Inputs:. Make sure you have your MIDI Out port selected under MIDI Outputs:. Under Port Mappings:, expand the entry for your MIDI Out port and make sure there is an entry saying MIDI-OX Events.

Click the OK button. Then select View->Input Monitor... to bring up the display of input messages. The main output monitor should already be displayed. If not, click the fifth icon on the toolbar, Main Output Monitor.

Turn your TrapKAT on and hit a pad once it's ready. You should see the MIDI message appear in MIDI-OX if you have everything set up right. If not, shout for help!

All data dump handling is done using MIDI-OX's SysEx tool, so do View->SysEx... to get there.

Receiving a MIDI Data Dump in MIDI-OX

Receiving a dump consists of two actions:

  1. Tell the receiver to listen.
  2. Tell the transmitter to send.

It's important to do it in that order or you'll miss the begining of the message!

In the MIDI-OX SysEx View and Scratchpad, do Sysex->Receive Manual dump to ready. Then on the TrapKAT, do Global footswitch, Pad 10, Pad 10 to do an All Memory dump. Once the TrapKAT completes, click the Done button on MIDI-OX. The Display Window will be full of numbers and letters representing the hexadecimal data that has been received.

This needs to be saved to disk before it can be loaded into the editor. Do Display Window->Save As and save the file somewhere you'll be able to find it later!

Transmitting a MIDI Data Dump in MIDI-OX

This is where life gets tricky.

The very first thing I suggest you do is configure the MIDI-OX Sysex tool to match the settings I use - if they don't work, then experiment...

Use Sysex->Configure to set the Low Level Output Buffers to:

  • Size: 32 bytes
  • Num: 512

and set Delay to 16 milliseconds between buffers. Click OK to save the changes.

It can be hard to tell if a dump has been successfully received if you haven't changed anything very obvious. My advice is to start by toggling the beeper on or off. Set it to zero for off, one for on. If you can reliably get this change to your TrapKAT, you should be able to rely on the timings.

Sending the file is as follows.

  1. Use Command Window->Load File... to choose the file you have edited (and saved!). The Command Window will be full of numbers and letters representing the hexadecimal data that has been loaded.
  2. Use Command Window->Send Sysex to transmit the data to the TrapKAT.
  3. Check the settings on the TrapKAT are as expected.

If you had no luck, you may need to adjust the timings.
